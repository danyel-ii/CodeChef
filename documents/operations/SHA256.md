# SHA-256

## What it does

SHA-256 maps any input to a 256-bit digest, rendered as 64 lowercase hexadecimal characters.

Among the hash operations currently in the app, this is the modern general-purpose option.

## Core algorithm

SHA-256 belongs to the SHA-2 family and processes 512-bit blocks.

1. Convert the input to bytes.
2. Append a `1` bit.
3. Pad with `0` bits until the message is 64 bits short of a block boundary.
4. Append the original message length as a 64-bit big-endian integer.
5. Initialize eight 32-bit state words.
6. For each block:
   - parse 16 words from the block
   - expand them to 64 words using the small sigma functions
   - run 64 rounds of compression
7. Concatenate the eight final state words.

## Compression details

Each round computes:

- `Ch(x,y,z) = (x AND y) XOR ((NOT x) AND z)`
- `Maj(x,y,z) = (x AND y) XOR (x AND z) XOR (y AND z)`
- two large sigma rotations on the running state
- one additive constant from the SHA-256 constant table

The algorithm repeatedly mixes message words into the 8-word state.

## Example

Input:

```text
hello
```

Digest:

```text
2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
```

## How the app behaves

- The app hashes the exact input bytes.
- It renders the digest as lowercase hexadecimal.
- The implementation uses Dart's `crypto` package.

## Security note

SHA-256 is a hash, not encryption. It is one-way by design and cannot be decoded back into the original message.
